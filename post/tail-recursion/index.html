What is tail recursion

A recursion function is that a function calls itself directly or indirectly. We ususally use recursion to address some problems like:

  • the solution can be implemented by recursion(e.g. backtracking).
  • the definition of data is fit to compute with recursion(e.g. Fibonacci).
  • the data structure can be organized or accessed by recursion(e.g. the traversal of binary tree).

Tail recursion is a concrete implement of recursion that just call the function itself in the end of codes. The tail recursive functions considered better than non-tail-recursive functions as tail-recursion can be optimized by compiler. The idea used by compilers to optimize tail-recursive functions is simple, since the recursive call is the last statement, there is nothing left to do in the current function, so saving the current functionâ€™s stack frame is of no use. Thus, complier can recognize the tail-recursion, replace the recursive call with goto statement and update the parameters adaptively.

// An example of tail recursive function 
void print(int n) 
{ 
    if (n < 0)   
       return; 
    cout << " " << n; 
  
    // The last executed statement is recursive call 
    print(n-1); 
} 

// Above code after tail call elimination 
void print(int n) 
{ 
start: 
    if (n < 0)  
       return; 
    cout << " " << n; 
  
    // Update parameters of recursive call 
    // and replace recursive call with goto 
    n = n-1 
    goto start; 
}

How to modify a recursion function to a tail one?

A tail recursion usually comes from a normal recursion function that execute the call function in the end of codes. To achieve this, we should change the local variables as the parameters. For example:

// recursion
function factorial(n) {
  if (n === 1) return 1;
  return n * factorial(n - 1);  // not just a function call
}

factorial(5) // 120

// tail recursion
function factorial(n, total) {
  if (n === 1) return total;
  return factorial(n - 1, n * total);
}

factorial(5, 1) // 120

In the above code blocks, the first one is a non-tail-recursive function to calculate factorial of n. Although it looks like a tail recursive at first look. If we take a closer look, we can see that the value returned by fact(n-1) is used in fact(n), so the call to fact(n-1) is not the last thing done by fact(n). Thus, in the second example, we use one more argument and accumulate the factorial value in second argument.
